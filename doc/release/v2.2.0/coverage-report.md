## Interpretation

- **Libraries (44.8% lines, 54.5% functions)** — the public C++ APIs that downstream
  users of MNE-CPP consume. Function coverage > 50% means the majority of exported
  entry points are exercised by at least one regression test. Linewise the gap is
  uncovered branches inside large pipeline classes (forward/inverse solvers,
  `MNEForwardSolution`, `Beamformer`, `DipoleFit`).
- **Applications (6.2%)** — almost entirely Qt UI / `mne_scan` / `mne_analyze` /
  `mne_browse` GUI code. Driving this through QtTest requires `offscreen` plus a
  composited Qt3D context which the headless coverage runner does not provide.
  These layers ARE covered by `mne_analyze_studio_workflow_integration`,
  `*viewer*` and `disp3d_*` tests on machines with a display, but those runs do
  not appear in this number.
- **Tools (35.2%)** — CLI conversion, anonymization and inverse helpers. Entry
  points and arg parsing covered (49% functions); the inner numerical loops are
  exercised indirectly via the matching `test_tool_*` testframes.

## Path to ≥ 50 % Overall

The 50% target is achievable but not in a single iteration on a headless host.
Plan:

1. **Wire missing sample-data tests** — the four SIGABRT failures are caused by
   missing or stale entries in `resources/data/mne-cpp-test-data/`. The data tree
   exists but a few subjects (`bem`, BEM model) need refreshing from the upstream
   `mne_cpp_test_data` repo. Fixing data unblocks ≈ 8 % more lines in `src/libraries/inv`
   and `src/libraries/fwd`.
2. **Run GUI/Qt3D tests on CI runner with X virtual framebuffer / `-platform offscreen`
   + `-graphicsapi software`**. This is the only way to lift `src/applications`
   above 6 %. macOS doesn't ship Xvfb; the existing GitHub Actions Linux job is
   the right place. We expect +12 % overall when the `*workflow*`,
   `*viewer*`, `*brainview*` tests run.
3. **Add data-driven testframes against mne-python references** (planned 20-40
   tests, naming pattern `test_<lib>_<feature>_vs_python_<scenario>`):
   - `test_dsp_firfilter_vs_python_bandpass_1_40Hz`
   - `test_inv_minimum_norm_vs_python_dSPM_evoked`
   - `test_fwd_bem_solution_vs_python_3layer_sphere`
   - `test_fiff_raw_io_vs_python_roundtrip_24bit`
   - `test_rtprocessing_averaging_vs_python_epochs_average`
   - `test_mne_anonymize_vs_python_meas_date_strip`
   - …
   Each test loads a `.npz`/`.fif` reference produced by mne-python, runs the
   corresponding C++ function and asserts `Eigen::isApprox(result, ref, 1e-6)`.
   Reference generators live in `src/testframes/generate_*_reference.py`.

## Reproducing

```bash
mkdir -p build_coverage && cd build_coverage
cmake -DCMAKE_BUILD_TYPE=Debug \
      -DCMAKE_CXX_FLAGS="--coverage -O0 -g" \
      -DCMAKE_EXE_LINKER_FLAGS="--coverage" \
      -DCMAKE_SHARED_LINKER_FLAGS="--coverage" \
      -DBUILD_TESTS=ON \
      -DCMAKE_PREFIX_PATH=$Qt6_DIR \
      ../src
cmake --build . -j8

cd ../out/Debug/tests
export DYLD_LIBRARY_PATH=$PWD/../lib
for t in test_*; do
    case "$t" in *viewer*|*workflow*|*brainview*|*brainsurface*|*disp3d*|*disp_plots*) continue;; esac
    gtimeout 600 "./$t" || echo "FAIL $t"
done

cd ../../../build_coverage
lcov --capture --directory . -o cov.info \
     --gcov-tool $(xcrun --find gcov) \
     --ignore-errors mismatch,negative,inconsistent,unused,format,gcov
lcov --remove cov.info '/Applications/*' '/Users/*/Qt/*' '*/external/*' '/usr/*' \
     '*/build_coverage/*' '*/testframes/*' '*moc_*' '*qrc_*' '*_autogen*' \
     -o cov_filtered.info \
     --ignore-errors unused,format,empty,inconsistent,negative,mismatch
genhtml cov_filtered.info -o coverage_html --quiet \
     --ignore-errors inconsistent,format,empty,unmapped,corrupt
```
