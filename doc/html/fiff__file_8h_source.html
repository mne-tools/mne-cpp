75 /*
76 * Conventions with FIFF_ C macros (mjk 14.01.2000).
77 *===========================================================================
78 * In order to get some order into the FIFF_ macros,
79 * following macro types are proposed:
80 *
81 * FIFF_ Fiff tag object identification label.
82 * FIFFB_ Fiff block tag value (block type indentification label).
83 * FIFFV_ Enumerated value of data having definite meaning in some context.
84 * Would be nice if the context is also present like:
85 * FIFFV_HPI_ACCEPT_PROGRAM and FIFFV_HPI_ACCEPT_USER.
86 * FIFFC_ Independent constant value like FIFFC_MATRIX_MAX_DIM used in
87 * code (having meaning in 'programming context' but not as data).
88 * FIFFT_ Fiff type descriptor.
89 * FIFFTS_ Fiff type decriptor (FIFFT_) structure definition (see below).
90 *
91 *
92 *
93 * Conventions on type codes: (mjk 14.01.2000)
94 *===========================================================================
95 * Fiff types are saved using 32 bit numeric indentifiers.
96 * The fiff type codes are structured so that they contain two main parts:
97 * the 'fundamental structure' and 'type details'. The fundamental structure
98 * is coded in the MSB of the 4 byte code. Depending on this code the
99 * interpretation of 'type details' may vary.
100 *
101 * Current Fundamental structures:
102 *----------------------------------------------------------------------
103 * Only the MSB is significant. See FIFFTS_FS_MASK
104 *
105 * FIFFFS_SCALAR 0x00000000 Scalar type / basic fixed size FIFF record.
106 * FIFFFS_RECORD 0x10000000 <reserved>
107 * FIFFFS_...... 0x20000000 <reserved>
108 * FIFFFS_...... 0x30000000 <reserved>
109 * FIFFFS_MATRIX 0x40000000 Multidimensional matrix.
110 *
111 * The lower four bits are reserved for future extensions.
112 *
113 * Scalar types (FS==0x00):
114 *----------------------------------------------------------------------
115 * These include the basic scalar types and 'standard' fixed size records
116 * used in fiff files.
117 *
118 * * It is required that the code is less than 0x0FFF (< 4096). !!!
119 * * Fourth byte is currently reserved for future extensions.
120 *
121 * Current types in this class are:
122 *
123 * FIFFT_VOID 0 Nothing
124 * FIFFT_BYTE 1 Unsigned? 8 bits
125 * FIFFT_SHORT 2 Signed 16 bit integer.
126 * FIFFT_INT 3 Signed 32 bit integer.
127 * FIFFT_FLOAT 4 Single precision IEEE float (32 bits)
128 * FIFFT_DOUBLE 5 Double precision IEEE float (64 bits)
129 * FIFFT_JULIAN 6 Julian day. (32 bits)
130 * FIFFT_USHORT 7 Unsigned short (16 bits)
131 * FIFFT_UINT 8 Unsigned int (32 bits)
132 * FIFFT_ULONG 9 Unsigned long (64 bits)
133 * FIFFT_STRING 10 Octet, ASCII coding.
134 * FIFFT_ASCII 10
135 * FIFFT_LONG 11 Long integer (64 bit)
136 * FIFFT_DAU_PACK13 13 13 bit packed format used in HP DAUs.
137 * FIFFT_DAU_PACK14 14 14 bit packed format used in HP DAUs
138 * FIFFT_DAU_PACK16 16 Signed 16 bit integer. (?)
139 * FIFFT_COMPLEX_FLOAT 20 Complex number encoded with floats
140 * FIFFT_COMPLEX_DOUBLE 21 Complex number encoded with doubles
141 * FIFFT_OLD_PACK 23 Neuromag proprietary 16 bit packing.
142 *
143 * Following are structure types defined in fiff_types.h
144 *
145 * FIFFT_CH_INFO_STRUCT 30 Basic info about a measurement chnnel.
146 * FIFFT_ID_STRUCT 31 Unique identifier.
147 * FIFFT_DIR_ENTRY_STRUCT 32 FIFF file directory entry.
148 * FIFFT_DIG_POINT_STRUCT 33 Digitization point.
149 * FIFFT_CH_POS_STRUCT 34 Channel position.
150 * FIFFT_COORD_TRANS_STRUCT 35 Coordinate transformation.
151 * FIFFT_DIG_STRING_STRUCT 36 Digitization string.
152 * FIFFT_STREAM_SEGMENT_STRUCT 37 Data stream segment.
153 * FIFFT_DATA_REF_STRUCT 38 Reference to external data.
154 *
155 * Simple vector of any scalar type can be formed by simply
156 * concatenating into file. The number of of elements is deduced
157 * from the size of the data block which should be a multiple of the
158 * object size.
159 *
160 * Futher more, for each scalar type fiff_types.h defines a corresponding
161 * C language type. Naming convention is fiff_xxx_t which correspons to
162 * type id FIFFT_xxx.
163 *
164 *
165 *
166 * Matrix types (FS=0x40)
167 *----------------------------------------------------------------------
168 *
169 * FIFF Matrix type is in principle an arbitrary dimensional rectangular
170 * collection of fixed size elements. However for practical reasons the
171 * maximum dimension is currenly restricted arbitrarily to 9
172 * (see FIFFC_MATRIX_MAX_DIM) but this restriction may be relaxed in the future.
173 *
174 * Matrix type codes use following structure:
175 *
176 * * Matrix type structure: 0xFFCCyyyy
177 * * Where
178 *
179 * 0xFF...... decribes the 'fundamental structure'.
180 * FF = 40 denotes a multidimensional matrix.
181 * 0x..XX.... describes to basic coding: dense, triangular, sparse etc.
182 * 0x....yyyy describes the element type.
183 *
184 * Current codings available are:
185 *
186 * FIFFTS_MC_DENSE 0x00000000 Dense column oriented matrix
187 * FIFFTS_MC_CCS 0x00100000 Column compressed sparse matrix
188 * FIFFTS_MC_RCS 0x00200000 Row compressed sparse matrix
189 *
190 *
191 * Dense matrix structure
192 * ----------------------
193 *
194 * A(1,1,...),A(2,1...) ... A(1,2,..) ... A(N,M,K...),K,M,N, ...,DIM
195 *
196 * where DIM is the dimensionality.
197 * N,M,K... are the dimensions.
198 * A(i,j,k...) are the elements.
199 *
200 * Note: the 2-dimensional meg_matrix format (routines in fiff_matrix.c)
201 * read and write the matrix in transposed form, i.e, the storing order
202 * in the FIFF-file is
203 * A(1,1),A(1,2) ... A(1,M) ... A(M,M),N,M,DIM
204 *
205 *
206 * Column compressed sparse matrix structure
207 *------------------------------------------
208 *
209 * A(x0,y0,z0...),A(x1,y0,z0),...
210 * ...,x0,x2...,x{NZ-1}, y0,y1,...,y{M-1}, z0,z1...,z{K-1},...,NZ,K,M,N,...,DIM
211 *
212 * where DIM is the dimensionality.
213 * N,M,K... are the dimensions.
214 * NZ is the number of non zero elements.
215 * A(i,j,k...) are the elements.
216 * x0..x(NZ-1) is row index array (concatenated for all vectors)
217 * y0..y(M-1) is column start index array for the second dimension.
218 * z1..z(K-1) is slice start index array for the third dimension.
219 * etc.
220 * Index arrays are 0 based.
221 *
222 * Row compressed sparse matrix structure
223 *---------------------------------------
224 *
225 * Similar to column compresses version except that dimensions 1 and 2 are
226 * interchanged. Structurally exactly same as ccs_matrix of the transpose.
227 *
228 */
