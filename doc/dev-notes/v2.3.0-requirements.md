## TASK 1 — mne-mna Python Package for MNA/MNX Interoperability

**Goal:** Create a Python package (`mne-mna`) that reads and writes `.mna`
(JSON) and `.mnx` (CBOR) files, integrating with MNE-Python workflows. This
enables Python-based researchers to consume and produce MNA project files,
bridging the MNE-CPP and MNE-Python ecosystems.

### Rationale for v2.3.0 (not v2.2.0)

- The MNA/MNX format should be published and citable (v2.2.0 TASK 17) before
  approaching the MNE-Python community for adoption.
- Coordination with MNE-Python maintainers is needed for API design review
  and potential joint publication.
- One more release cycle allows the format to stabilize based on internal
  usage and feedback.

### Package scope

#### Core I/O

| Function | Description |
|----------|-------------|
| `mne_mna.read_project(path)` | Read `.mna` (JSON) or `.mnx` (CBOR) → `MnaProject` object |
| `mne_mna.write_project(project, path)` | Write `MnaProject` → `.mna` or `.mnx` (auto-detected by extension) |
| `mne_mna.mna_to_mnx(in_path, out_path)` | Convert JSON ↔ CBOR |

#### Data model (Python classes mirroring C++)

| Class | Maps to C++ |
|-------|-------------|
| `MnaProject` | `MNALIB::MnaProject` |
| `MnaGraph` | `MNALIB::MnaGraph` |
| `MnaNode` | `MNALIB::MnaNode` |
| `MnaPort` | `MNALIB::MnaPort` |
| `MnaSubject` | `MNALIB::MnaSubject` |
| `MnaSession` | `MNALIB::MnaSession` |
| `MnaRecording` | `MNALIB::MnaRecording` |
| `MnaFileRef` | `MNALIB::MnaFileRef` |
| `MnaScript` | `MNALIB::MnaScript` |
| `MnaVerification` | `MNALIB::MnaVerification` |
| `MnaParamTree` | `MNALIB::MnaParamTree` |
| `MnaParamBinding` | `MNALIB::MnaParamBinding` |
| `MnaOpSchema` | `MNALIB::MnaOpSchema` |

Enums: `MnaDataKind`, `MnaFileRole`, `MnaNodeExecMode`, `MnaPortDir` as
Python `enum.Enum` classes with identical string mappings.

#### MNE-Python integration

| Feature | Description |
|---------|-------------|
| `project.to_raw(node_id)` | Extract `mne.io.Raw` from a project's file refs |
| `project.to_epochs(node_id)` | Extract `mne.Epochs` |
| `project.to_source_estimate(node_id)` | Extract `mne.SourceEstimate` |
| `project.add_raw(raw, subject, session, recording)` | Add an `mne.io.Raw` as a file ref |
| `MnaGraph.to_networkx()` | Export DAG as a NetworkX graph for visualization |
| `MnaGraph.from_networkx(G)` | Import DAG from NetworkX |

#### BIDS integration

| Feature | Description |
|---------|-------------|
| `mne_mna.from_bids(bids_path)` | Create `MnaProject` from an existing BIDS dataset (using `mne_bids`) |
| `mne_mna.to_bids(project, bids_root)` | Export file refs into BIDS directory structure |

### Dependencies

- `cbor2` — CBOR encoding/decoding (for `.mnx`)
- `mne` — MNE-Python (optional; required for `to_raw()` etc.)
- `mne_bids` — MNE-BIDS (optional; required for BIDS integration)

### Package structure

```
mne-mna/
  pyproject.toml
  README.md
  LICENSE
  src/
    mne_mna/
      __init__.py
      _project.py      # MnaProject, MnaSubject, MnaSession, etc.
      _graph.py         # MnaGraph, MnaNode, MnaPort
      _io.py            # read_project, write_project, mna_to_mnx
      _types.py         # Enums (MnaDataKind, MnaFileRole, etc.)
      _script.py        # MnaScript
      _verification.py  # MnaVerification, MnaProvenance
      _params.py        # MnaParamTree, MnaParamBinding
      _schema.py        # MnaOpSchema
      _mne_bridge.py    # to_raw, to_epochs, etc. (optional mne dep)
      _bids_bridge.py   # from_bids, to_bids (optional mne_bids dep)
  tests/
    test_io.py
    test_roundtrip.py
    test_mne_bridge.py
    test_bids_bridge.py
    conftest.py
```

### Cross-validation with MNE-CPP

- Round-trip test: C++ writes `.mna` → Python reads → Python writes → C++
  reads → verify identical.
- Same for `.mnx` (CBOR).
- Use `resources/examples/` MNA files as shared test fixtures.

### Publication coordination

- Contact MNE-Python core team (Gramfort, Larson, Engemann) after v2.2.0
  release with the published format paper DOI.
- Propose `mne-mna` as a companion package (similar to `mne-bids`,
  `mne-connectivity`, `mne-icalabel`).
- Joint publication opportunity: MNA/MNX as a reproducible analysis container
  format for the MNE ecosystem.

### Acceptance criteria

1. `pip install mne-mna` installs cleanly on Python 3.10+.
2. Round-trip `.mna` read/write preserves all fields.
3. Round-trip `.mnx` read/write preserves all fields including embedded binary data.
4. `MnaProject` ↔ MNE-Python object conversion works for Raw, Epochs, and
   SourceEstimate.
5. `from_bids()` correctly imports a BIDS dataset into MNA project structure.
6. All enum string mappings match C++ implementation exactly.
7. CI cross-validates Python ↔ C++ round-trips on every commit.
